Experiment 4 - Pyramids in World Space

A collection of cartoony pyramids floating in a colorful space. The user can control their camera to get a different perspective as the pyramids break off from the pack to go their separate ways (WASD to move horizontally, Q/E to move vertically, IJKL to rotate).

Documentation

First, I wanted to modify the code so that it was easier to read and understand. I started by removing bits of code that were unnecessary to making the artpiece work like the “kumaleon” class that would store the canvas despite it not being used in the code. Commenting out the “applyMatrix” line adjusted the “perspective” of the scene, which I thought was interesting.

After substantial truncation, I managed to trim the code down to a small snippet of what I felt was important enough to serve as a steppingstone for my experiment. In its current state, the code creates a canvas, then creates the shapes to occupy the canvas space. I also adjusted the colors so that they could be more “predictable” as the original code chose from an abstract table of values that didn’t exactly explain the functionality of the artwork.

I then attempted to apply a sine-wave motion to the drawing, but instead of moving each individual shape, it simply squashed and stretched the already-existing image in a bobbing motion. While the results were not what I anticipated, the test still helped me visualize how objects in 3D space could move around and how I could use the matrix function to apply shifts and rotations to the result.

Documentation

I first started by converting the canvas to WEBGL to enable 3D, then I created an array of 3D pyramid shapes on the canvas.

I felt that the artwork wasn’t visually interesting enough, so I added functionality to rotate the screen based on the mouse’s location, and I adjusted the spacing and color of the shapes to better contrast with the background.

While experimenting, I thought of the idea of allowing the user to control a virtual camera much like in a game engine, using the arrow keys for controlling the camera orientation and using the WASD keys to move around the scene.

After adjusting the “feel” of the camera movement, I then adjusted the amount and movement of the shapes such that they will start to float away in a random direction over time. To emphasize the 3D effect of the scene, I added the ability to pan the camera up and down with the “Q” and “E” keys. I also added a colored background that changes upon each launch.

To add some “juice” to the experiment, I added a “cartoon” aesthetic to the shapes and gave them a darker side to give the illusion of “shadows” in the scene. I also adjusted the spacing of the shapes to give each one more prominence.

After playing around with the experiment for a little while, I decided to add smooth camera movement to emphasize the “weight” of the user’s actions and to make the experience feel more polished. I also changed the camera rotation input from the arrow keys to “IJKL” respectively; the arrow keys did not work well with websites as they would try to navigate within the page as well as the canvas’ camera.

  • Contributions

    I made the entirety of the code for this artwork, originally taking inspiration from work done by the inspirers mentioned above. I utilized the methods the “KUMALEON Coding Challenge Sample” used to create the 3D shapes such as the matrix rotation to allow for camera movement.

  • Reflection

    I think this experiment, despite being relatively simple on the surface, turned out to be quite a fun challenge in terms of having to change my workflow from a 2D environment into a 3D space with an extra dimension and the difficulties that come with navigating in more than two dimensions.

  • Highs

    I enjoy the ability for the use to “navigate” within the canvas by using keyboard inputs, allowing them to observe the movement of the pyramids in real-time from a variety of perspectives. I also like the “smooth” camera movements that make the navigation feel “weighty” and fluid.

  • Lows

    I couldn’t get a good start on this experiment because of my illness from last week, which also made it difficult to find an interesting idea for an experiment based in 3D graphics. In addition, performance costs made it difficult to add more than a set number of distinct shapes to the canvas without causing degradation.
